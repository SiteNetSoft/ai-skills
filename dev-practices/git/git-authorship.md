# Git Authorship — Human-Only Attribution

When performing any git operation (commit, rebase, amend, merge, tag, etc.), **never include references to AI, Claude, assistants, copilots, or any non-human entity** in:

- `Co-Authored-By` trailers
- `Signed-off-by` trailers
- Commit message bodies or footers
- Author or committer fields
- Tag annotations
- PR/MR descriptions generated alongside commits

## Rules

1. **No AI co-author lines.** Never append lines like:
   ```
   Co-Authored-By: Claude <noreply@anthropic.com>
   ```
   The human user is the sole author.

2. **No AI mentions in commit messages.** Do not include phrases like "generated by AI", "written with Claude", "assisted by Copilot", or similar in any part of the commit message.

3. **Use the repository's configured git identity.** Do not set or override `user.name`, `user.email`, or `GIT_AUTHOR_*` / `GIT_COMMITTER_*` environment variables. Let the existing git config determine authorship.

4. **Respect existing commit style.** Before writing a commit message, read recent `git log --oneline` output and match the repository's conventions (prefix tags, casing, tense, length). The only change from normal behavior is the removal of AI attribution — everything else stays the same.

5. **Applies to all git porcelain.** This covers `git commit`, `git merge --no-ff`, `git tag -a`, `git rebase`, `git cherry-pick`, interactive fixups, squash merges, and any other command that produces a commit object.

## What to do instead

- Write clear, concise commit messages that describe **what** changed and **why**.
- Follow the project's existing commit conventions.
- Attribute human collaborators with `Co-Authored-By` when appropriate.
- Focus the commit message on the change, not on how it was produced.
